## 0. 一句话定位

把 **真实 NVIDIA PTX 子集**（`.version 7.0 / sm_70`）编译成 **AEC 128-bit 定长机器码 `.aecbin`**，正确性是门禁，周期数（`total_cycles`）越低性能分越高。

**评分（100）**：正确性 50（按类别加权 **T1×4 + T2×8 + T3×10 + T4×12 + T5×16**）+ 性能 35（T2:5 / T3:9 / T4:10 / T5:11）+ 鲁棒 5 + Agent 10。
**性能归一化**：`r=T_base/T_你`，`p(r)=clip((log r−log0.5)/(log2−log0.5),0,1)` → 追平基线 50%、2× 满分。

## 1. ⚠️ 五个必须先知道的硬约束（踩中任何一个 = 正确性直接挂）

### 1.1 编码必须用 Track-B §A.1（C1 的 ISA 权威是赛题 B）
C1 spec.md 明确「请从赛题B中获得ISA的spec」，所以二进制编码以 **Track-B spec.md §A.1 opcode 表 + §4 type 表** 为准。用 Track-B `testcases/tests/aec_cases/*/program.bin`（实际执行的二进制）解码验证如下：

| | **Track-B §A.1/§4（C1 正确）** | C2 `b_isa_public.json`（仅 C2,勿用） |
|---|---|---|
| CVTFF/CVTFI/CVTIF/CVTII | **0x50/0x51/0x52/0x53** | 0x53/0x54/0x55/0x56 |
| CPY | **0x54** | 0x51 |
| LOADI / LOADI64 | **0x55 / 0x56** | 0x50 / 0x52 |
| type f32 / f16 | **0x8 / 0xa** | 0 / 2 |
| type u32 / b32 | **0x2 / 0x0** | 8 / 13 |
| type none | **0xf**（`.none` 指令类型位就是 0xf） | 0xf |

验证：Track-B `cvtff` 测例 `CPY.u32` → `word3=0x00540010`（opcode=0x54,type u32=2）,`CVTFF.f16.f32` → type 位=f16(0xa)@[6:3] + src f32(0x8)@[13:10]。**`include/aec/isa.h` + `src/isa/encoder.cpp` 已按 Track-B 编号,`sim/aec_decode.py` 同步,selfTest 8/8（向量取自 Track-B `cvtff/program.bin`）。端到端验证:12/12 个自包含 Track-B aec_cases 在我们 sim 上执行结果与 `expected/gmem` 逐字节一致。**
👉 **法则:语义 + 编码都以 Track-B spec.md 附录 A 为准。** C2 的 `b_isa_public.json`/`aec_isa.h` 是 C2 专有的另一套编号（LOADI=0x50/CPY=0x51/f32=0）,**与 C1 无关,禁止使用**。fp8/fp4/s4/u4 在 §4 是保留位,AEC 不存在。

### 1.2 AEC 没有 64-bit 整数加法 —— 地址运算要塌缩成 32-bit
`ADD/SUB/MUL` 合法类型只有 `.u32/.s32/.f32/.f16/.bf16/.f64`，**没有 `.u64/.b64`**。但所有 PTX 都用 `add.u64`/`mul.wide.u32` 做地址。
关键事实：**ISA 规定"所有架构地址均为对应 space 内的 unsigned 32-bit byte offset"**（Track-B §5 / §8.1）。所以：
- `ld.param.u64` 取到的基址，其**低 32 位就是 gmem byte offset**，高 32 位无意义。
- `mul.wide.u32 %rd, %r, 4`（32×32→64）对现实索引不会溢出 32 位 → 当作 `MUL.u32`（取低 32）。
- `add.u64 %rd, base, off` → **`ADD.u32`（或 `.b32`）只算低字**。
👉 lowering 把所有指针类型 `.b64` vreg 当 32-bit 处理，只保留低字。脚手架现在 objdump 里出现的 `ADD.b64`（见 line 12-14 的反汇编）**是非法指令**（ADD 无 b64），这是已知正确性缺口，必须改。

### 1.3 divergent 分支 `@p bra` 不能直接编成 BRX —— 要 if-conversion 谓词化
AEC 是"launch 决定 active mask + lockstep"模型，**无 SIMT 重汇聚栈**。`BRX` 要求**所有 active lane 条件一致**（mixed → execution error）。
但 PTX 里的边界保护 `setp.ge.u32 %p, tid, n; @%p bra DONE;` 在 **N 不是 blockDim 整数倍**时是发散的（部分 lane 越界退出）→ 直接 `BRX` 会 execution error。
👉 **必须把边界保护做 if-conversion（谓词化）**：算出 `p = (tid < n)`，把 guard 之后到 `DONE` 的所有指令都加谓词 `@p`（谓词假的 lane 无副作用，天然正确）。这正是评分里的"**谓词执行优化**"。
- **循环回边**（`@p bra LOOP`，trip count 是编译期常量 32/16/K，各 lane 相同）→ **uniform，可以用 BRX**。
- 只有"线程发散的条件"才谓词化；uniform 的循环/跳转用 BR/BRX。
- PTX-01 N=4096÷256=16 整除，公开测试碰巧不发散；但**鲁棒性测试会改 N**（非整除）→ 不谓词化就挂。

### 1.4 f16 没有 2-byte load —— T5 的隐藏坑
`LD` 合法宽度只有 4/8 字节（`.b32/.b64/...`），**没有 `.u16`**。但 PTX-05 用 `ld.global.u16` 逐个取 f16 元素。
两条路：
- **标量回退**：`LD.b32`（4 字节含 2 个 f16）后按 `addr&2` 决定 `SHR 16` + `AND 0xffff` 抽出目标 f16，再 `CVTFF f32<-f16`。可行但繁琐，且要保证 4 字节不越界。
- **张量路径（推荐拿 perf）**：整段 K-loop 降到 `TLDA/TMUL/TSTA`，tile load 原生按 f16 处理，绕开逐元素 2-byte load。

### 1.5 `mad.f32` 应映射到 `FMA`（单次舍入），不是 `MAD`
sm_70 的 PTX `mad.f32` 是**融合乘加（单次舍入 = fma）**；而 AEC `MAD` 语义是"乘法先舍入再加法舍入"（Track-B §7.1），`FMA` 才是无限精度单次舍入。
👉 **PTX `mad.f32/fma.rn.f32` → AEC `FMA.f32`**；**PTX `mad.lo.u32` → AEC `MAD.u32`**（整数无舍入问题）。搞错 → GEMM 累加 fp 不 bit-exact → T5 正确性挂。

## 2. PTX 子集全集（parser 必须覆盖的指令）

5 个文件用到的**全部** PTX（隐藏测试是这些的变异，不会超纲太多）：

| PTX 指令 | → AEC lowering |
|---|---|
| `.version/.target/.address_size/.visible .entry/.param/.reg` + label + `{}` | 指令流/符号 |
| `ld.param.{u64,u32,f32}` | `LD.pmem.{b64→b32,u32,f32}`（低字；param block 按声明序自然打包） |
| `mov.u32 %r,%tid.x/%ctaid.x/%ntid.x/%ctaid.y` | `CPY.u32 Rd,%special` |
| `mov.{u32,f32} %r, imm` | `LOADI Rd, imm32` |
| `mad.lo.u32` | `MAD.u32` |
| `mul.wide.u32 %rd,%r,imm` | `MUL.u32`（低 32，需先 LOADI 常量） |
| `add.u64` / `add.u32` | `ADD.u32`（低字，见 1.2） |
| `mul.lo.u32` | `MUL.u32` |
| `and.b32 %r,%r,imm` / `shr.u32 %r,%r,imm` | `AND/SHR.u32`（**AEC 无立即数形式** → 先 `LOADI` 到寄存器再 reg-reg） |
| `setp.{ge,lt,...}.u32 %p,%r,%r` | `CMPP.{ge,lt}.u32 Pd,Rs1,Rs2` |
| `@%p bra LABEL` | 发散→谓词化；uniform 循环→`BRX P,->pc`（见 1.3） |
| `add/sub/mul.f32` | `ADD/SUB/MUL.f32` |
| `mad.f32` | **`FMA.f32`**（见 1.5） |
| `cvt.f32.f16 %f,%r` | `CVTFF`（dst f32, src f16） |
| `ld.global.u16` | 见 1.4（2-byte load 合成 或 走 TMUL） |
| `ld.global.f32` / `st.global.f32` | `LD.gmem.f32` / `ST.gmem.f32` |
| `ret` | `RET`（kernel 尾部；单 warp 模型下 RET/HALT 结束） |

**立即数是通用难点**：AEC 只有 `LOADI/LOADI64/BR/BRX/CALL` 带立即数，算术/逻辑/移位全是 reg-reg。→ 需要一个**立即数物化 pass**：把 PTX 里的常量（`4`、`31`、`0`、`16`、循环上界等）`LOADI` 到 vreg，常量传播 + CSE 再消掉重复。

## 3. 五个测试类别逐一拆解（考什么 / 什么 pass 拿分）

| 类别 | PTX | 正确性权重 | 性能权重 | 核心考点 | 拿分的 pass |
|---|---|---:|---:|---|---|
| **T1** 基础 lowering | PTX-01 vadd | 4 | 0 | 全流程跑通、编码合法 | parser→IR→encode，**谓词化边界**、**64位塌缩** |
| **T2** 控制/标量 | PTX-02 invariant_poly | 8 | 5 | 循环不变量、CSE、DCE | **LICM**（把 `a+b`/`a*b` 提出 32 次循环）+ **CSE**（`%f5==%f6`）+ **DCE**（`%f15` 死值）+ 常量传播 |
| **T3** 内存 | PTX-03 repeated_reuse | 10 | 9 | load 复用、SMEM 提升 | **LICM on load**（`[%rd5]` 地址循环不变 → 提出 16 次循环）+ 内存合并 + smem 缓存 |
| **T4** 寄存器/调度 | PTX-04 reg_schedule | 12 | 10 | 活跃区间、DDG、双发射 | **linear-scan 寄存器分配** + **list scheduling**（多个独立 load 提前发射、隐藏 32-cycle 延迟）+ **双发射配对** |
| **T5** Tensor/GEMM | PTX-05 gemm_f16 | **16** | 11 | TMUL 下降、tiling、多精度 | **GEMM 惯用法识别 → TLDA/TMUL/TSTA** + tiling + 多精度（9 种）+ 余数 tile |

**关键 ROI 观察**：
- **T5 权重 16（正确性）最高**。即使**标量正确**的 f16 GEMM（不做 TMUL）也能拿 16 正确性分；TMUL 只加 11 性能分。→ 先做"标量正确 GEMM"落袋 16 分，再上 TMUL 抢 11。
- T2+T3 是**纯标量小 IR**，LICM/CSE/DCE 是教科书算法，性价比极高（14 性能分 + 18 正确性分）。
- 正确性是**perf 门禁**：任何类别编译结果跑错，该类别的性能分直接归零。所以 **correctness-first**。

## 4. 完整实现流程（分阶段 · 映射脚手架文件 · 对应知识库）

现有脚手架已完成 **Phase 0**（parser/IR/encoder ✅bit-exact/binfmt/objdump/driver 全通，`make` 过，5 个 PTX 都能编译反汇编）。**passes 是恒等占位**，真正拿分从 Phase 1 起。

### Phase 1 — T1-T4 正确性（解锁 34 正确性分 + 全部性能门禁）【最高优先】
按 1.2-1.5 修 lowering 正确性，都在 `src/codegen/lower.cpp` + `src/ir/ir_builder.cpp`：
1. **发散分支谓词化（if-conversion）** —— 边界 guard → 谓词；uniform 循环留 BRX。（`ir_builder.cpp` / 新增 `passes/pred_opt.cpp` 真实化）
2. **64-bit 地址塌缩 32-bit** —— `.b64` 指针只留低字，`add.u64→ADD.u32`。
3. **立即数物化** —— PTX 常量 `LOADI` 到 vreg（`ir_builder.cpp`）。
4. **`mad.f32→FMA`，`mad.lo.u32→MAD`** 的正确映射。
5. **param block 布局** —— 声明序自然打包（objdump 已这么做）；⚠️ 具体偏移要匹配 C1 golden 期望，列入 confirm。

### Phase 2 — T2/T3 标量优化（14 性能 + 巩固 18 正确性）【高 ROI】
`src/passes/*.cpp` 真实化，`driver.cpp` 里 -O2/-O3 迭代到定点：
- `const_prop.cpp` + `dce.cpp`（杀 `%f15`）+ `cse.cpp`（合并 `%f5/%f6`）+ `licm.cpp`（提循环不变量/不变 load）。
- 知识库：标准数据流分析（到达定义、可用表达式、循环不变量检测）；需要先建**支配树 + 自然循环识别**（`cfg.cpp` 扩展）。

### Phase 3 — T4 寄存器分配 + 调度（10 性能）
- `regalloc/linear_scan.cpp`：活跃区间 + 线性扫描分配 256 GPR + 溢出到 `.lmem`。知识库：**Poletto & Sarkar 线性扫描**（[SSA 版带代码](https://bernsteinbear.com/blog/linear-scan/)）。
- `sched/list_sched.cpp`：建 DDG（真依赖/反依赖/输出依赖）+ list scheduling（关键路径优先，load 提前）+ 双发射配对（相邻无依赖指令配对）。知识库：List Scheduling + [寄存器分配/调度综述](https://arxiv.org/pdf/1409.7628)。注意寄存器分配与调度相互影响（先调度还是先分配，建议 pre-RA 调度）。

### Phase 4 — T5 GEMM→TMUL（16 正确性 + 11 性能，单项最大）
`codegen/gemm_tmul.cpp`：
1. **识别 GEMM 惯用法**：K-loop 内 `acc = fma(a[..],b[..],acc)`，acc 是循环携带的规约变量（脚手架已有 detection scaffold）。
2. **先做标量正确 GEMM**（含 1.4 的 f16 load 合成 + FMA 累加）→ 落袋 16 正确性分。
3. **再降到张量指令**：`TLDA` tile-load A、`TLDA` tile-load B、`TMUL`（mode: f16=1，见 docs/03 §6）、`TSTA` tile-store；tiling 16×16。
4. **多精度参数化**：隐藏测试覆盖 fp4/fp8(e4m3/e5m2)/f16/bf16/f32/int4/int8/int32（TMUL mode 表见 docs/03），及非 16 倍数 M/N/K → **余数 tile 掩码**。
5. 知识库：[im2col/Winograd 不需要]，GEMM tiling 经典分块；精度语义看 C2 `docs/04_数值规范.md`。

### Phase 5 — 鲁棒性（5）+ Agent（10）
- **鲁棒性**：50 个变异（改 N、寄存器重命名、BB 重排、循环次数、死代码、寄存器压力、**PTX-05 改 dtype**、内存复用变化）。Phase 1 的谓词化 + 通用 regalloc + dtype 参数化 GEMM 覆盖大部分；**严禁按文件名/ID 硬编码**。
- **Agent**（`agent/run_agent.py`）：8 分 = 在自己默认输出上 geomean 加速 ≥1.25×；2 分 = 闭环完整（独立跑/读报告/重编译/验证/生成报告）。⚠️ 只有 Phase 2-4 的 pass 变"真"之后 Agent 才有加速空间；**闭环 2 分可先白拿**（脚手架已有 sweep 框架）。

## 5. 🔑 最高杠杆的一次性投资：自建 AEC 功能模拟器

**C1 不发独立 golden / cycle model,但 Track-B `testcases/tests/aec_cases/` 提供 36 个 `program.bin` + 35 个 `expected/gmem` 执行 golden** —— 既能反解真实编码,又能把它们喂进我们的 sim 验证执行语义(12/12 自包含用例逐字节一致)。50 正确性分不再是盲飞。

强烈建议花半天写一个 **~300-500 行的 AEC 功能解释器**（`tools/aec-sim`）：
- 读 `.aecbin`，按 128-bit 解码，模拟 256 GPR + 8 pred + gmem/pmem + 单 warp（甚至单 thread 先行），执行到 RET。
- 用 numpy/C 算 vadd/poly/gemm 参考结果，对拍解释器输出。
- 这样 T1-T5 的**执行正确性本地可验**，否则改任何 lowering 都不知道对不对。
- 参考知识库理解执行模型：[GPGPU-Sim](https://github.com/gpgpu-sim/gpgpu-sim_distribution) / [Vortex](https://github.com/vortexgpgpu/vortex) / [tiny-gpu](https://github.com/adam-maj/tiny-gpu)。cycle 估计可在解释器上叠加简单延迟模型（LD/LMEM 32 cycle、算术 1、SFU 若干），对齐 `est_cycles`。

> 没有这个 oracle，Phase 1-4 的每次改动都无法验证正确性 —— 这是 C1 最该先补的工具。
