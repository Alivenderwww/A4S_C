# C3.5 推理后端改造：onnxruntime → CuPy 手写算子

## 目标
响应 spec 新要求（第 9 行"数值计算库统一采用 CuPy"），把 C3.5 推理后端从 onnxruntime 改成 **CuPy 手写 17 个算子在 GPU 上执行**。范围限定 C3.5（C3.3 的 ops_numpy 保持 numpy，它是融合对齐的 CPU 参考实现，不涉及性能分）。

## 改造范围（3 个文件）
1. **新建 `runtime/ops_cupy.py`** — 17 个算子的 CuPy 实现，签名逐个对照 `ops_numpy.py`
2. **新建 `runtime/cupy_runtime.py`** — CuPy 图执行器，结构镜像 `mock_runtime.py`
3. **改 `tools/infer.py`** — 新增 `_CupyBackend`，IO/manifest/批切/写出全部复用

## 实现细节

### ops_cupy.py（对照 ops_numpy 的 17 算子，按三模型实测 attrs 实现）
| 算子 | CuPy 实现 | 难点/注意 |
|---|---|---|
| Relu/Add/Sub/Mul/Div/Sqrt | `cp.maximum`/运算符/`cp.sqrt` | 直接替换 |
| MatMul | `cp.matmul`（支持批量） | transformer 性能热点，25 次 |
| Gemm | transB→swapaxes + matmul + bias | 三模型均 transB=1,alpha=beta=1 |
| Flatten | reshape | axis=1 |
| **Conv** | **im2col + matmul** | 用 `cp.lib.stride_tricks.as_strided` 替换 numpy 三重循环；group=1（三模型无 depthwise） |
| GlobalAveragePool | `cp.mean(axis=(2,3))` | |
| LayerNormalization | mean/var/sqrt 逐元素 | axis=-1, 注意用均值估方差非 np.var |
| Softmax | 数值稳定版手写 | axis=-1 |
| Gather | `cp.take` | int64 索引（transformer 词嵌入） |
| Split | `cp.split` + cumsum | axis=-1，3 输出 |
| Transpose | `cp.transpose` | 显式 perm |
| Reshape | reshape | 处理 -1 和 0 |
| **Erf** | `cupyx.scipy.special.erf` | transformer GELU 用，CuPy 无原生 erf |
| Constant | 保留 numpy（多为 host 元数据） | shape/split 参数不上 GPU |

### cupy_runtime.py（CupyRuntime 类，镜像 MockRuntime）
- 构造时：`import_onnx_graph` 加载图，**所有 float32 initializer 预上 GPU 并缓存**（权重只传一次），int64 initializer（shape/split 元数据）保留 numpy
- `run(feeds)`：feeds 上 GPU → `graph.topo_order()` 逐算子执行 → 输出 `cp.asnumpy` 回 numpy
- 复用 `scheduler.Graph.topo_order()` 的拓扑排序
- Constant/Gather 的 int64 索引路径特殊处理（不上 GPU 或按需）

### infer.py 改造
- 新增 `_CupyBackend`：`__init__` 调 `import_onnx_graph` + `CupyRuntime`，`run` 调 `CupyRuntime.run`
- `_make_backend` 改为**优先 CuPy**，onnxruntime 作为 fallback（本机无 GPU 或 CuPy 装不上时仍能跑）
- **IO/manifest/批切/写出逻辑全部保留不动**（已与后端解耦）

## 前置：安装 CuPy
本机有 GTX 1650 Ti（4GB）+ CUDA。执行阶段先装：`pip install cupy-cuda12x`（按 nvidia-smi 的 CUDA 版本选 cuda11x/12x）。验证 `cp.cuda.runtime.getDevice()` 可用。

## 验证标准（对照 spec C3.5）
1. **精度门槛**（15 分，硬要求）：三模型 `np.allclose(out, golden, rtol=1e-3, atol=1e-3)` 全过；MLP top1≥98%、ResNet top1≥85%
2. **逐算子对齐**：每个 CuPy 算子与 numpy 参考做 max_abs_diff < 1e-5（transformer 之前 numpy 版 diff=8.2e-6，CuPy 版要对齐到同等水平）
3. **selftest 回归**：35/35 仍通过（C3.1 不受影响）
4. **显存**：GTX 1650 Ti 只有 4GB，ResNet 输入 10000×3×32×32≈122MB，需靠 batch_size 分批控制峰值（现有分批机制已支持）

## 关键风险与对策
- **Erf 精度**：`cupyx.scipy.special.erf` 需单独验证与 math.erf 的差异（fp32 下）。若超阈，用有理逼近自写
- **Conv im2col 显存**：as_strided 是零拷贝视图，不增显存；但 im2col 展开后矩阵 `(N, C*kh*kw, oh*ow)` 可能大，需确认 batch_size 下不爆 4GB
- **GTX 1650 Ti 4GB 显存约束**：ResNet 是最大模型，权重 ~44MB + 激活，batch=256 可能偏大，必要时降到 64-128

## 不做的事
- C3.3 的 ops_numpy 不改 CuPy（CPU 参考实现，融合对齐用，不需 GPU）
- C3.1/C3.2/C3.4 不涉及数值计算，不动
- 不删 onnxruntime（保留为 fallback，防御性）
