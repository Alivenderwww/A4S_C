## 2. ⚠️ 本机环境实况（务必先读，有坑）

我在本机（Windows 11）探测的结果：

| 工具 | 本机状态 | 影响 |
|---|---|---|
| g++（PATH） | **4.9.2（2014，太旧，连 `-std=c++17` 都不认）** | 评测机是 GCC 13.3。**C2 本地根本编不了**（需现代 g++/WSL）；C1 已做兼容（Makefile 自动回退 `-std=c++11`），本机可编可跑 |
| make | 4.4.1 ✅ | 可用 |
| Python | PATH 的 `python`=3.12（**无 pip**）；**`py -3.13` 才是可用环境** | C3 一律用 `py -3.13`（已装好 onnx 1.22 / numpy 2.3 / onnxruntime 1.27） |
| onnx / numpy / onnxruntime | **`py -3.13` 已装** ✅ | C3.1 + C3.5 正确性已在本机 CPU 跑通（selftest 35/35） |
| torch / GPU | **无** | C3.5 的运行时间/显存排名分**无法在本机测**，需一台 CUDA GPU 机器 |

**三条硬结论：**
1. **C2 无法在原生 Windows 构建** —— `libaec_device.so` 是 Linux ELF，且本机 g++ 4.9.2 连 `-std=c++17` 都不认。**必须用 WSL 或 Linux**（现代 g++）跑 `make -j2` 才能编译与自测。（代码本身已 signature-complete，本机唯一报错是继承自 starter-kit 的无消息 `static_assert`，在 GCC 13.3 下合法）
2. **C1 本机可编可跑**（Makefile 自动回退 c++11，已验证：`make` 通过、selftest 8/8、PTX-01 编出 20 条指令且 objdump 回读正常）；正式以评测机 GCC 13.3 为准，联调建议也用 Linux/WSL。
3. **C3 用 `py -3.13`**：C3.1+C3.5 正确性本机 CPU 已跑通（ResNet top1 93.5%、Transformer diff 3.9e-5）；C3.5 的**性能排名分需 GPU 机器**。

> 建议：尽早统一到一台 Linux（或每人本机 WSL）+ 一台带 NVIDIA GPU 的机器（供 C3.5 联调）。

## 3. 各人 5 分钟上手

### C1（编译器 · 你）— ✅ 已验证可编可跑
```bash
cd C1
make                 # 本机 g++ 也能编（自动回退 c++11）；Windows 产物是 bin/aec-cc.exe
make selftest        # 编码器 8 个 golden 向量 bit-exact
make test            # 编译全部 5 个公开 PTX（21/21 检查）
bin/aec-cc ../public/Track-C/C1-compiler/testcases/PTX-01_vector_add.ptx -O2 -o out.aecbin
bin/aec-objdump out.aecbin
```
先看 `C1/README.md` 的架构图与 TODO 清单。**优先级按正确性权重**：T5(TMUL/GEMM)×16 > T4×12 > T3×10 > T2×8 > T1×4。当前 passes 是恒等占位（r=1.0），真正拿分从 `codegen/gemm_tmul.cpp`(T5) 与 `sched/list_sched.cpp`+`regalloc/linear_scan.cpp`(T4) 入手。

### C2（Runtime · 队友）
```bash
# 必须在 WSL/Linux
cd C2
make -j2 && make examples
./bin/01_device_query
python3 grader/public_grade.py --submission . --profile public   # 自测打分
python3 cases/test_r101.py --submission .                         # 单项
```
先看 `C2/README_C2实现说明.md`。**优先级**：Basic 全过（R101-R106 + FP32/INT32 GEMM）→ Good（stream/event/异步DMA/注册内存/故障恢复）→ Excellent（两个 Agent）。

### C3（调度器 · 队友）— ✅ C3.1+C3.5 正确性已跑通（selftest 35/35）
```bash
cd C3
py -3.13 -m pip install -r requirements.txt   # 本机 py -3.13 已装好，无需重装
# C3.1 图解析（10分，最稳）
py -3.13 tools/export_dag.py --onnx ../public/Track-C/C3-scheduler/testcases/release_to_competitors/models/mlp_v1.onnx --output dag.json
# C3.5 端到端（先保正确性过门禁）
py -3.13 tools/infer.py --onnx <model> --input <input_dir> --output <out_dir> --batch-size 256
# 自测（35/35 通过）+ C3.2/3.3 自评
py -3.13 tests/selftest_c31_c35.py
py -3.13 benchmarks/c32_c33/bench_c32_c33.py --models mnist_mlp cifar_resnet18 transformer
```
先看 `C3/README.md`。**优先级**：C3.1(10)+C3.5 正确性(15) 已到手 → C3.3 补 Conv-BN 预融合(F1 第5分) → C3.5 GPU 性能(时间25+显存10) → C3.2 微调。
