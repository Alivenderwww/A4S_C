## 二、已验证的硬性数字

### L1 — 官方 golden 自检（基于官方公开素材）

> ⚠️ 以下数字均为**本地结果**，非官方隐藏评测。

| 项目 | 结果 |
|------|------|
| `make selftest`（encoder golden 8/8） | ✅ 全部逐位一致（基于官方公开 golden 向量） |
| `make test`（5 公开 PTX 编译 + objdump） | ✅ 21/21（每个用例多步检查，基于官方公开 PTX） |

### L3 — 自建 simulator 回归（bench.py，基于自建 AEC sim，非官方 golden）

| 指标 | 计数值 |
|------|-------:|
| **PASS** | **132** |
| DIVERGE | 0 |
| FAIL | 0 |
| SIM-ERR | 0 |
| COMPILE-ERR | 0 |

> **重要**：这些是**自建 AEC 功能模拟器上的回归**，验证的是"编译器产出与 numpy 独立参考一致"。**不是官方隐藏评测**，不能等同于官方的 132 题通过。simulator 本身的语义若有偏差，PASS 可能掩盖真正的 bug。
>
> bench 覆盖：参数/矩阵尺寸扫描、寄存器重命名、基本块重排、循环次数变更、死代码插入、寄存器压力增加、GEMM dtype 变更（bf16/f32）、内存复用模式。
>
> 四类非 PASS 计数全为 0（DIVERGE/FAIL/SIM-ERR/COMPILE-ERR），共 5 类指标。

### 回归已抓出的真实 bug（4 个）

1. **R5 跨循环 clobber**：reuse 用例——旧线性扫描不跨回边延伸区间。
2. **CVT 源类型未编码**：encoder 只编 dst type，与 golden 不匹配。
3. **跨块 CSE**：非 SSA 假名导致错误复用。
4. **b64 寄存器对 clobber**：寄存器压力变异暴露——allocator 不为 pair 保留 Rd+1。

| 模块 | 状态 | 风险 |
|------|------|------|
| **const_prop** | 🔶 identity stub | 常量映射框架，无折叠/传播（returns false） |
| **mem_coalesce** | 🔶 identity stub | 扫描并计数 load/store，无改写（returns false） |
| **gemm_tmul** (TMUL 下降) | 🔶 仅检测 | `looksLikeGemmLatch` 识别 GEMM 惯用法，**不生成 TLDA/TMUL/TSTA IR**；多精度/TMUL 均未实现 |
| **spill** (线性扫描) | 🔶 **钳位桩** | 寄存器超限时钳位到 R255/R254 并计数 `spillCount++`，**不插入真实 ST.lmem/LD.lmem**，可能静默算错 |
| **driver：多 kernel** | 🔶 只编译 `functions[0]` | 多 entry 被忽略 |
| **f16 2-byte load** | 🔶 **原始字节地址 LD 降级** | `ld.global.u16` 直降为原始字节地址 `LD.gmem.b32`（读 4 字节），CVT 取低 16 位；**无 4B 对齐+addr&2 移位提取**。**末元素在 gmem 末尾多读 2 字节**（flat gmem 一般无害，边界敏感）；奇数半字地址语义也有风险。P0 指向 `ir_builder.cpp` |

| **fp8/fp4/int GEMM** | ❌ 未实现 | 依赖 TMUL 下降 + 多精度参数化 |

## 七、限制与风险（完整清单）

### 自建 oracle 的边界

1. **不是官方 golden**：sim 语义与真正 golden 若有出入，sim PASS ≠ 官方通过。
2. **单 warp 顺序**：无 atomics/barrier/smem 跨线程建模（公开 kernel 用不到）。
3. **FP 精度**：FMA 用 float64 中间量模拟单次舍入；MAD 两次舍入。
4. **BRX 发散检测**：只验证 uniform；正确实现应无 DIVERGE（已谓词化）。

### 工程风险

1. **identity stub 静默通过**：const_prop/mem_coalesce 返回 false（无改变），不会报错，但可能错过正确性和性能分。
2. **spill 钳位可能算错**：超寄存器时映射到同一物理寄存器 = 冲突，可能产生错误结果。
3. **f16 边界越界**：`ld.global.u16` 降为原始字节地址 `LD.gmem.b32`（读 4 字节），末元素在 gmem 末尾多读 2 字节；奇数半字地址语义也有风险。
4. **三套本地性能代理不一致**：Agent 已知输入用 sim scoreboard（LD=32）选优，未知用 driver est_cycles（LD=8）；scheduler 延迟表仅影响调度就绪列表排序，不直接作为 Agent 测量源。DEFAULT_CONFIG 硬编码 O0 可能不符合官方默认配置。加速比计算跨口径时失真。
5. **CVT 仍可能漏**：已补源类型编码，但隐藏测试的精度组合可能覆盖不足。
6. **多 kernel 未实现**：若隐藏测试有多个 entry 函数，当前只编译第一个。
7. **依赖 `public/` 路径**：`Makefile` 和测试脚本引用 `../public/Track-C/`，需确保路径在各环境一致。
